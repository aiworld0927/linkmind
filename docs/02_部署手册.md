# LinkMind 部署手册

## 一、环境准备

### 1.1 基础要求

| 环境 | Python | Neo4j | Docker | 内存 |
|------|--------|-------|--------|------|
| 开发 | 3.12+ | 5.26.0+ | 可选 | 4GB+ |
| 测试 | 3.12+ | 5.26.0+ | 推荐 | 8GB+ |
| 生产 | 3.12+ | 5.26.0+ | 必须 | 16GB+ |

### 1.2 依赖安装工具

推荐使用 `uv` 作为 Python 包管理器：

```bash
# 安装 uv (Linux/macOS)
curl -LsSf https://astral.sh/uv/install.sh | sh

# 安装 uv (Windows)
powershell -c "irm https://astral.sh/uv/install.ps1 | iex"
```

## 二、开发环境部署

### 2.1 独立进程运行

#### 步骤一：启动 Neo4j 数据库

**方式一：使用 Docker（推荐）**

```bash
# 启动 Neo4j 容器
docker run -d \
  --name neo4j-dev \
  -p 7687:7687 \
  -p 7474:7474 \
  -e NEO4J_AUTH=neo4j/password \
  -e NEO4J_dbms_memory_pagecache_size=512M \
  -e NEO4J_dbms_memory_heap_max__size=1G \
  neo4j:5.26.0

# 检查运行状态
docker ps | grep neo4j

# 查看日志
docker logs -f neo4j-dev
```

**方式二：本地安装 Neo4j**

1. 下载地址：https://neo4j.com/download/
2. 安装完成后启动：
   ```bash
   # Linux
   sudo systemctl start neo4j

   # macOS
   brew services start neo4j

   # Windows
   neo4j console
   ```

#### 步骤二：配置环境变量

创建或修改 `backend/.env` 文件：

```env
# Neo4j 连接配置
NEO4J_URI=bolt://localhost:7687
NEO4J_USER=neo4j
NEO4J_PASSWORD=password
```

#### 步骤三：安装项目依赖

```bash
cd backend

# 创建虚拟环境并安装依赖
uv pip install -e .

# 验证依赖安装
uv pip list
```

#### 步骤四：启动开发服务器

```bash
cd backend/src

# 方式一：直接运行（支持热重载）
python main.py

# 方式二：使用 uvicorn 启动
uvicorn main:app --host 0.0.0.0 --port 8000 --reload

# 方式三：使用 uv run
uv run python main.py
```

**启动成功输出：**

```
INFO:     Will watch for changes in these directories: ['/coding/linkmind/backend/src']
INFO:     Uvicorn running on http://0.0.0.0:8000 (Press CTRL+C to quit)
INFO:     Started reloader process [12345] using StatReload
INFO:     Started server process [12347]
INFO:     Waiting for application startup.
INFO:     Application startup complete.
```

## 三、测试环境部署

### 3.1 使用 Docker Compose

#### 步骤一：配置测试环境变量

创建 `deploy/.env.test` 文件：

```env
# Neo4j 配置
NEO4J_AUTH=neo4j/testpassword
NEO4J_dbms_memory_pagecache_size=1G
NEO4J_dbms_memory_heap_max__size=2G

# 后端配置
NEO4J_URI=bolt://neo4j:7687
NEO4J_USER=neo4j
NEO4J_PASSWORD=testpassword
```

#### 步骤二：启动测试环境

```bash
cd deploy

# 构建并启动所有服务
docker-compose --env-file .env.test up -d --build

# 查看服务状态
docker-compose ps

# 查看日志
docker-compose logs -f

# 查看特定服务日志
docker-compose logs -f backend
```

#### 步骤三：验证服务

```bash
# 检查健康状态
curl http://{服务器IP}:8000/api/v1/health

# 检查图谱数据
curl http://{服务器IP}:8000/api/v1/graph/

# 检查 API 文档
curl http://{服务器IP}:8000/docs
```

> **说明**：将 `{服务器IP}` 替换为实际的服务器IP地址，例如 `192.168.1.100`

### 3.2 测试环境配置说明

| 服务 | 端口 | 说明 |
|------|------|------|
| backend | 8000 | API 服务 |
| neo4j | 7687 | 图数据库 |
| neo4j-web | 7474 | Neo4j 控制台 |

### 3.3 停止测试环境

```bash
cd deploy

# 停止服务
docker-compose down

# 停止服务并删除数据卷（谨慎使用）
docker-compose down -v
```

## 四、生产环境部署

### 4.1 使用 Docker Compose

#### 步骤一：配置生产环境变量

创建 `deploy/.env.prod` 文件：

```env
# Neo4j 配置
NEO4J_AUTH=neo4j/your_strong_password
NEO4J_dbms_memory_pagecache_size=4G
NEO4J_dbms_memory_heap_max__size=8G

# 后端配置
NEO4J_URI=bolt://neo4j:7687
NEO4J_USER=neo4j
NEO4J_PASSWORD=your_strong_password
```

**安全注意事项：**
- 使用强密码（12位以上，包含大小写字母、数字、特殊字符）
- 不要将密码提交到版本控制系统
- 定期更换密码

#### 步骤二：优化 Dockerfile（生产模式）

修改 `backend/Dockerfile` 以支持生产环境：

```dockerfile
# 生产环境 Dockerfile
FROM python:3.12-slim as builder

WORKDIR /app

COPY pyproject.toml .

RUN pip install --no-cache-dir .

FROM python:3.12-slim

WORKDIR /app

COPY --from=builder /usr/local/lib/python3.12/site-packages /usr/local/lib/python3.12/site-packages
COPY --from=builder /usr/local/bin /usr/local/bin

COPY . .

WORKDIR /app/src

# 生产模式使用多个 worker
CMD ["uvicorn", "main:app", "--host", "0.0.0.0", "--port", "8000", "--workers", "4"]
```

#### 步骤三：启动生产环境

```bash
cd deploy

# 使用生产配置启动
docker-compose --env-file .env.prod up -d

# 查看服务状态
docker-compose ps
```

### 4.2 使用 Gunicorn 部署（推荐）

修改 Dockerfile 使用 Gunicorn：

```dockerfile
FROM python:3.12-slim

WORKDIR /app

COPY pyproject.toml .

RUN pip install --no-cache-dir . \
    && pip install gunicorn

COPY . .

WORKDIR /app/src

CMD ["gunicorn", "-w", "4", "-k", "uvicorn.workers.UvicornWorker", "-b", "0.0.0.0:8000", "main:app"]
```

### 4.3 Nginx 反向代理配置

创建 `/etc/nginx/conf.d/linkmind.conf`：

```nginx
server {
    listen 80;
    server_name your-domain.com;

    # 前端页面
    location /frontend {
        proxy_pass http://localhost:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }

    # API 接口
    location /api/ {
        proxy_pass http://localhost:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_read_timeout 300s;
    }

    # API 文档
    location /docs {
        proxy_pass http://localhost:8000;
    }
}
```

### 4.4 生产环境安全建议

1. **防火墙配置**：只开放必要端口（80, 443）
2. **HTTPS**：使用 Let's Encrypt 配置 SSL 证书
3. **备份策略**：定期备份 Neo4j 数据目录
4. **监控告警**：配置 Prometheus + Grafana 监控
5. **日志管理**：使用 ELK 栈收集和分析日志

## 五、独立进程运行（不使用 Docker）

### 5.1 安装 Neo4j

#### Ubuntu/Debian

```bash
# 添加 Neo4j 仓库
wget -O - https://debian.neo4j.com/neotechnology.gpg.key | sudo apt-key add -
echo 'deb https://debian.neo4j.com stable 5.26' | sudo tee -a /etc/apt/sources.list.d/neo4j.list

# 安装 Neo4j
sudo apt update
sudo apt install neo4j=1:5.26.0

# 启动服务
sudo systemctl enable neo4j
sudo systemctl start neo4j

# 验证安装
neo4j status
```

#### macOS

```bash
# 使用 Homebrew 安装
brew install neo4j@5.26

# 启动服务
brew services start neo4j@5.26

# 验证安装
neo4j status
```

#### Windows

1. 下载安装包：https://neo4j.com/download/
2. 运行安装程序
3. 启动 Neo4j 服务

### 5.2 配置 Neo4j

编辑 `/etc/neo4j/neo4j.conf`（Linux）或 `conf/neo4j.conf`（Windows/macOS）：

```conf
# 允许远程访问
dbms.default_listen_address=0.0.0.0

# 配置内存
dbms.memory.heap.max_size=8G
dbms.memory.pagecache.size=4G

# 配置认证
dbms.security.auth_enabled=true
```

### 5.3 启动后端服务

```bash
cd backend

# 创建虚拟环境
python -m venv .venv

# 激活虚拟环境
source .venv/bin/activate  # Linux/macOS
.venv\Scripts\activate     # Windows

# 安装依赖
pip install -e .

# 启动服务
cd src
python main.py
```

## 六、快速启动脚本

### 6.1 开发环境启动脚本

创建 `scripts/start-dev.sh`：

```bash
#!/bin/bash

echo "=== 启动 LinkMind 开发环境 ==="

# 检查 Docker 是否运行
if ! docker info > /dev/null 2>&1; then
    echo "错误: Docker 未运行，请先启动 Docker"
    exit 1
fi

# 启动 Neo4j
echo "1. 启动 Neo4j 数据库..."
docker run -d \
  --name neo4j-dev \
  -p 7687:7687 \
  -p 7474:7474 \
  -e NEO4J_AUTH=neo4j/password \
  neo4j:5.26.0

echo "等待 Neo4j 启动..."
sleep 10

# 安装依赖
echo "2. 安装项目依赖..."
cd backend
uv pip install -e .

# 启动后端服务
echo "3. 启动后端服务..."
cd src
python main.py
```

## 七、常见问题

### Q1: Neo4j 连接失败

**原因**：数据库未启动或连接配置错误

**解决方案**：

```bash
# 检查 Neo4j 是否运行
docker ps | grep neo4j

# 如果未运行，启动 Neo4j
docker start neo4j-dev

# 检查连接配置
cat backend/.env

# 测试连接
neo4j-admin database check neo4j
```

### Q2: 端口被占用

**原因**：8000 或 7687 端口已被其他服务占用

**解决方案**：

```bash
# 查找占用端口的进程
lsof -i :8000
lsof -i :7687

# 终止占用进程
kill -9 <PID>
```

### Q3: 依赖安装失败

**原因**：网络问题或 Python 版本不兼容

**解决方案**：

```bash
# 使用国内镜像源
uv pip install -e . --index-url https://pypi.tuna.tsinghua.edu.cn/simple

# 检查 Python 版本
python --version
```

### Q4: 前端页面无法访问

**原因**：后端服务未启动或路径配置错误

**解决方案**：

```bash
# 检查后端服务状态
curl http://localhost:8000/

# 检查前端文件路径
ls -la frontend/index.html
```

## 九、备份与恢复

### 9.1 Neo4j 数据备份

```bash
# 停止写入
docker exec neo4j-dev neo4j-admin database stop neo4j

# 创建备份
docker exec neo4j-dev neo4j-admin database backup \
  --to-path=/backup \
  neo4j

# 启动数据库
docker exec neo4j-dev neo4j-admin database start neo4j
```

### 9.2 Neo4j 数据恢复

```bash
# 停止数据库
docker exec neo4j-dev neo4j-admin database stop neo4j

# 恢复数据
docker exec neo4j-dev neo4j-admin database restore \
  --from-path=/backup/neo4j \
  neo4j

# 启动数据库
docker exec neo4j-dev neo4j-admin database start neo4j
```

## 十、升级指南

### 10.1 版本升级步骤

1. **备份数据**（见第九章）
2. **停止服务**：`docker-compose down`
3. **拉取最新代码**：`git pull`
4. **更新依赖**：`uv pip install -e .`
5. **重启服务**：`docker-compose up -d`
6. **验证服务**：`curl http://localhost:8000/api/v1/health`

### 10.2 数据库迁移

如果数据模型有变更：

```bash
# 执行数据库初始化（重建索引）
docker exec -it backend bash -c "python -c \"from database.init_indexes import init_database_indexes; import asyncio; asyncio.run(init_database_indexes())\""
```
