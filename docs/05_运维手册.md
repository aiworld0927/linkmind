# LinkMind 运维手册

## 一、磁盘管理

### 1.1 磁盘膨胀原因分析

开发过程中虚拟机磁盘从 20GB 膨胀到 120GB 的主要原因：

| 原因 | 占比 | 说明 |
|------|------|------|
| Docker 镜像缓存 | ~60% | 多次构建和拉取镜像，旧镜像未清理 |
| Docker 容器 | ~5% | 已停止但未删除的容器占用空间 |
| Docker 本地卷 | ~10% | Neo4j 数据卷和其他持久化存储 |
| Node.js 依赖 | ~15% | `node_modules` 和 `.node` 目录 |
| Python 虚拟环境 | ~5% | `.venv` 目录 |
| 构建缓存 | ~3% | Vite/Webpack 构建缓存 |
| 系统日志 | ~2% | 长时间运行产生的日志 |

### 1.2 磁盘清理步骤

#### 步骤一：清理 Docker 资源（最有效）

```bash
# 查看 Docker 磁盘使用情况
docker system df

# 清理所有未使用的资源（推荐）
docker system prune -a

# 清理未使用的卷（谨慎使用，会删除数据卷）
docker volume prune

# 删除特定未使用的镜像
docker rmi <IMAGE_ID>

# 删除已停止的容器
docker rm $(docker ps -a -q)

# 删除所有悬空镜像
docker rmi $(docker images -f "dangling=true" -q)
```

#### 步骤二：清理 Node.js 依赖

```bash
# 清理前端 node_modules（重新安装时会重建）
rm -rf frontend/node_modules

# 清理 npm 缓存
npm cache clean --force

# 清理 .node 目录（如果不再需要）
rm -rf .node
```

#### 步骤三：清理 Python 虚拟环境

```bash
# 清理后端虚拟环境（重新安装时会重建）
rm -rf backend/.venv
```

#### 步骤四：清理构建缓存

```bash
# 清理 Vite 构建缓存
rm -rf frontend/.vite

# 清理前端构建产物（开发环境不需要）
rm -rf frontend/dist
```

#### 步骤五：清理系统日志

```bash
# 清理系统日志
sudo journalctl --vacuum-size=100M

# 清理 apt 缓存
sudo apt clean

# 清理旧内核（谨慎使用）
sudo apt autoremove --purge
```

### 1.3 VMware 虚拟机磁盘压缩

#### 方法一：在虚拟机内清理后压缩（推荐）

1. **在虚拟机内执行清理**（见上文步骤）
2. **填充空闲空间为零**：
   ```bash
   sudo dd if=/dev/zero of=/tmp/zeros bs=1M
   sudo rm /tmp/zeros
   ```
3. **关闭虚拟机**
4. **在 VMware Workstation 中压缩**：
   - 选择虚拟机 → 右键 → 管理 → 清理磁盘
   - 或使用命令行：
     ```bash
     vmware-vdiskmanager -k "/path/to/your/virtual_machine.vmdk"
     ```

#### 方法二：使用 VMware 压缩工具

```bash
# Windows
"C:\Program Files (x86)\VMware\VMware Workstation\vmware-vdiskmanager.exe" -k "D:\VMs\linkmind.vmdk"

# Linux
vmware-vdiskmanager -k "/var/lib/libvirt/images/linkmind.qcow2"

# macOS
/Applications/VMware\ Fusion.app/Contents/Library/vmware-vdiskmanager -k "/Users/username/Documents/Virtual Machines/linkmind.vmdk"
```

### 2.3 备份策略

#### Neo4j 数据备份

```bash
# 创建备份目录
mkdir -p /backup/neo4j

# 停止数据库
docker-compose stop neo4j

# 创建备份
docker run --rm \
  -v linkmind_neo4j_data:/data \
  -v /backup/neo4j:/backup \
  neo4j:5.26.0 \
  neo4j-admin database backup --to-path=/backup neo4j

# 启动数据库
docker-compose start neo4j
```

#### 自动备份脚本

创建 `scripts/backup.sh`：

```bash
#!/bin/bash

BACKUP_DIR="/backup/neo4j/$(date +%Y%m%d)"
mkdir -p $BACKUP_DIR

echo "=== 开始备份 Neo4j 数据 ==="

# 停止写入
docker exec linkmind-neo4j-1 neo4j-admin database stop neo4j

# 创建备份
docker exec linkmind-neo4j-1 neo4j-admin database backup \
  --to-path=$BACKUP_DIR \
  neo4j

# 启动数据库
docker exec linkmind-neo4j-1 neo4j-admin database start neo4j

# 清理7天前的备份
find /backup/neo4j -type d -mtime +7 -exec rm -rf {} \;

echo "=== 备份完成 ==="
echo "备份路径: $BACKUP_DIR"
```

配置定时任务：

```bash
crontab -e

# 添加每日凌晨2点备份
0 2 * * * /coding/linkmind/scripts/backup.sh >> /var/log/linkmind_backup.log 2>&1
```

## 三、故障排查

### 3.1 常见问题

#### Q1: 服务无法启动

**原因**：端口被占用、配置错误、依赖缺失

**解决方案**：

```bash
# 检查端口占用
lsof -i :8000

# 检查配置文件
cat backend/.env

# 检查 Docker 日志
docker-compose logs backend

# 检查 Neo4j 连接
neo4j-admin database check neo4j
```

#### Q2: 数据库连接失败

**原因**：Neo4j 未启动、认证失败、网络问题

**解决方案**：

```bash
# 检查 Neo4j 状态
docker-compose ps neo4j

# 重启 Neo4j
docker-compose restart neo4j

# 检查认证配置
docker exec neo4j-dev neo4j-admin dbms set-initial-password <new_password>
```

#### Q3: 前端页面无法访问

**原因**：Vite 服务未启动、端口配置错误、API 代理问题

**解决方案**：

```bash
# 检查 Vite 服务
curl http://localhost:5173

# 检查代理配置
cat frontend/vite.config.ts

# 重启前端服务
cd frontend && npm run dev
```

#### Q4: 内存溢出

**原因**：Neo4j 内存配置不足、查询过于复杂

**解决方案**：

```bash
# 增加 Neo4j 内存（修改 docker-compose.yml）
NEO4J_dbms_memory_heap_max__size=4G
NEO4J_dbms_memory_pagecache_size=2G

# 重启服务
docker-compose up -d
```

### 3.2 紧急恢复

```bash
# 停止所有服务
docker-compose down

# 恢复备份
docker run --rm \
  -v linkmind_neo4j_data:/data \
  -v /backup/neo4j/latest:/backup \
  neo4j:5.26.0 \
  neo4j-admin database restore --from-path=/backup neo4j

# 启动服务
docker-compose up -d

# 验证恢复
curl http://localhost:8000/api/v1/graph/
```

## 四、资源优化

### 4.1 Docker 优化

```bash
# 使用 BuildKit 构建（更快、更小）
export DOCKER_BUILDKIT=1
docker-compose build

# 清理构建缓存
docker builder prune

# 使用多阶段构建（减小镜像体积）
# 参考 backend/Dockerfile
```

### 4.2 Neo4j 优化

```bash
# 配置内存（根据服务器配置调整）
# docker-compose.yml
NEO4J_dbms_memory_heap_max__size=4G
NEO4J_dbms_memory_pagecache_size=2G

# 配置线程数
NEO4J_dbms_threads_worker_count=8
NEO4J_dbms_threads_lock_manager=4

# 配置日志级别
NEO4J_dbms_logs_level=WARN
```

### 4.3 前端优化

```bash
# 生产构建（压缩代码）
cd frontend && npm run build

# 启用 gzip 压缩（在 nginx 中配置）
# /etc/nginx/conf.d/linkmind.conf
gzip on;
gzip_types text/html text/css application/javascript application/json;
```

## 五、版本更新

### 5.1 更新流程

1. **备份数据**（见第二章）
2. **拉取最新代码**：
   ```bash
   git pull origin main
   ```
3. **更新依赖**：
   ```bash
   cd backend && uv pip install -e .
   cd frontend && npm install
   ```
4. **构建前端**：
   ```bash
   cd frontend && npm run build
   ```
5. **重启服务**：
   ```bash
   docker-compose down
   docker-compose up -d --build
   ```
6. **验证服务**：
   ```bash
   curl http://localhost:8000/api/v1/health
   ```

### 5.2 回滚流程

```bash
# 回滚代码
git log --oneline -5
git checkout <COMMIT_HASH>

# 重启服务
docker-compose down
docker-compose up -d --build

# 验证回滚
curl http://localhost:8000/api/v1/health
```

    scrape_interval: 15s

### 6.2 配置 Grafana（可选）

1. 导入 Neo4j 监控仪表盘
2. 创建自定义告警规则：
   - CPU 使用率 > 80%
   - 内存使用率 > 85%
   - 磁盘使用率 > 90%
   - 服务响应时间 > 500ms
