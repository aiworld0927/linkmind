```typescript
// 循环依赖检测
checkCircleDepend: (nodeList, newDependIds, selfId) => {
  const visited = new Set();
  const stack = [...newDependIds];
  
  while (stack.length > 0) {
    const currentId = stack.pop();
    if (!currentId) continue;
    if (currentId === selfId) return true;
    if (visited.has(currentId)) continue;
    visited.add(currentId);
    
    const currentNode = nodeList.find(n => n.id === currentId);
    if (currentNode) {
      stack.push(...currentNode.dependencies);
    }
  }
  return false;
}
```

### 3.5 知识下钻

**功能描述**：双击节点进入下一层级，展示该节点的附属笔记转换的子节点。

**技术实现**：

| 双击事件 | X6 node:dblclick 事件 | [GraphCanvas.tsx:507-522](file:///coding/linkmind/frontend/src/components/GraphCanvas.tsx#L507-L522) |
| 层级切换 | Zustand drillDown | [store/index.ts:274-308](file:///coding/linkmind/frontend/src/store/index.ts#L274-L308) |
| 面包屑导航 | Ant Design Breadcrumb 组件 | [Breadcrumbs.tsx](file:///coding/linkmind/frontend/src/components/Breadcrumbs.tsx) |
| 层级回退 | Zustand backToBreadLevel | [store/index.ts:310-346](file:///coding/linkmind/frontend/src/store/index.ts#L310-L346) |

**核心代码示例**：

```typescript
// 层级下钻
drillDown: (targetNodeId) => {
  const node = state.currentNodes.find(n => n.id === targetNodeId);
  if (!node) return;
  
  // 保存当前层级到历史
  const historyEntry = {
    levelId: state.currentLevelId,
    nodes: state.currentNodes.map(n => ({ ...n })),
    breadcrumbs: [...state.breadcrumbs],
  };
  
  // 将笔记转换为子节点
  const childNodes = node.children.map(child => convertListItemToNodeItem(child));
  
  // 更新状态
  set({
    currentLevelId: targetNodeId,
    breadcrumbs: [...state.breadcrumbs, { id: node.id, name: node.name }],
    currentNodes: childNodes,
    levelHistory: [...state.levelHistory, historyEntry],
  });
}
```

## 四、开发规范

### 4.1 前端开发规范

**组件开发**：
- 使用函数式组件 + Hooks
- 组件命名使用 PascalCase
- 文件命名使用 PascalCase
- 使用 TypeScript 接口定义 props

**状态管理**：
- 全局状态使用 Zustand
- 组件内部状态使用 useState
- 计算属性使用 useMemo
- 回调函数使用 useCallback

**代码风格**：
- 使用 TypeScript 严格模式
- 使用 `const` 而非 `var`
- 避免未使用的变量和导入
- 使用 `===` 而非 `==`

### 4.2 后端开发规范

**API设计**：
- RESTful 风格
- 使用统一的响应格式 `{ code, msg, data }`
- 使用 HTTP 状态码
- 参数校验使用 Pydantic

**数据库操作**：
- Repository 层封装数据库操作
- 使用参数化查询防止注入
- 事务操作使用 `session.run`

**代码风格**：
- 使用 Python 3.12+ 语法
- 使用类型提示
- 使用 `async/await` 异步编程
- 函数命名使用 snake_case

## 六、故障排查

### 6.1 前端常见问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| 页面白屏 | Vite 模块解析错误 | 检查 importmap 配置 |
| 节点不显示 | X6 容器尺寸为0 | 确保容器有明确的宽高 |
| 拖拽无效 | 事件被拦截 | 检查 event.stopPropagation |
| 连线消失 | 节点 ID 不匹配 | 检查数据源和节点 ID |

### 6.2 后端常见问题

| 问题 | 原因 | 解决方案 |
|------|------|----------|
| Neo4j 连接失败 | 数据库未启动 | 启动 Docker Neo4j 容器 |
| API 500 错误 | 未捕获异常 | 查看后端日志 |
| 索引未创建 | 首次启动未初始化 | 检查 `init_database_indexes` |
