# LinkMind 前端功能 PRD

## 一、项目背景

当前前端为单HTML文件实现，功能简单，仅支持基础图谱可视化。为提升用户体验，需要构建完整的前端工程，实现笔记分级分层展示、节点编辑交互、时序关系拖拽等高级功能。

## 二、目标用户

知识工作者、学习者、研究人员，需要管理和探索个人知识库的用户。

## 三、核心功能需求

### 功能一：分级分层显示笔记内容

**用户故事：**
> 作为知识管理者，我希望笔记内容能够分级分层显示，这样可以清晰地看到笔记的结构和层次关系。

**需求描述：**
- 笔记内容支持 Markdown 格式渲染
- 支持标题分级（H1-H6）
- 支持列表、引用、代码块等格式
- 内容过长时支持折叠展开
- 支持目录导航，点击可跳转

**用户流程：**
1. 用户点击图谱中的节点
2. 系统在侧边栏展示笔记详情
3. 笔记内容按 Markdown 格式渲染，标题自动分级显示
4. 长内容自动折叠，用户可点击展开
5. 右侧目录导航支持点击跳转

**验收标准：**
- [ ] Markdown 语法正确渲染（标题、列表、代码、引用）
- [ ] 标题分级清晰，字体大小递减
- [ ] 内容超过300字自动折叠，显示"展开全文"按钮
- [ ] 目录导航根据标题自动生成
- [ ] 点击目录项可平滑跳转到对应位置

### 功能二：节点展示、跳转和编辑

**用户故事：**
> 作为知识编辑者，我希望点击节点后可以查看详情、跳转到关联节点、编辑内容，这样可以高效管理知识图谱。

**需求描述：**
- 点击节点显示详情侧边栏
- 详情包含标题、内容、元数据（创建时间、社区ID、度数）
- 支持跳转到关联节点
- 支持编辑节点标题和内容
- 编辑后自动保存到后端

**用户流程：**
1. 用户点击图谱中的节点
2. 右侧弹出详情侧边栏
3. 用户可查看完整笔记内容
4. 用户点击"编辑"按钮进入编辑模式
5. 修改标题和内容后点击"保存"
6. 系统自动同步到后端，图谱实时更新

**验收标准：**
- [ ] 点击节点1秒内弹出侧边栏
- [ ] 侧边栏显示完整笔记信息
- [ ] 编辑模式支持修改标题和内容
- [ ] 保存成功后提示用户
- [ ] 图谱节点实时更新显示

### 功能三：拖拽编辑节点时序关系

**用户故事：**
> 作为时间线管理者，我希望能够拖拽调整节点间的时序关系，这样可以直观地管理知识的时间顺序。

**需求描述：**
- 支持切换到时序视图模式
- 节点按时间顺序排列在时间线上
- 支持拖拽节点调整顺序
- 拖拽完成后自动更新节点的时序属性
- 支持设置节点间的依赖关系

**用户流程：**
1. 用户点击"时序视图"按钮切换模式
2. 图谱切换为时间线布局
3. 用户拖拽节点到新位置
4. 系统实时更新时间线显示
5. 用户释放鼠标后保存新的时序关系

**验收标准：**
- [ ] 时序视图模式切换流畅
- [ ] 节点按时间顺序排列
- [ ] 拖拽操作响应灵敏
- [ ] 释放后自动保存时序关系
- [ ] 依赖关系箭头正确显示

## 四、技术方案

### 技术栈

| 技术 | 版本 | 说明 |
|------|------|------|
| Vue | 3.4+ | 前端框架 |

│   └── Common/
│       ├── Toolbar.vue          # 工具栏
│       ├── SearchPanel.vue      # 搜索面板
│       └── Toast.vue            # 提示组件

│   └── GraphView.vue            # 主视图
├── stores/
│   ├── graph.js                 # 图谱状态管理

├── services/

### 状态管理设计

#### Graph Store
- nodes: 节点列表
- links: 关系列表
- selectedNode: 当前选中节点
- highlightedNodes: 高亮节点集合
- viewMode: 视图模式（graph/timeline）

#### Note Store
- currentNote: 当前笔记详情
- isEditing: 是否编辑模式
- contentPreview: 内容预览状态

#### Timeline Store
- timelineNodes: 时序节点列表
- dependencies: 依赖关系列表
- isDragging: 是否拖拽中

## 五、API接口需求

### 现有接口（可复用）
- GET /api/v1/graph/ - 获取图谱数据
- GET /api/v1/notes/{id} - 获取笔记详情
- PUT /api/v1/notes/{id} - 更新笔记
- GET /api/v1/expand/neighborhood - 展开邻居

### 新增接口需求
- PUT /api/v1/notes/{id}/order - 更新节点时序顺序
- GET /api/v1/graph/timeline - 获取时序数据

## 六、UI/UX设计

### 配色方案
- 主背景：#0f172a（深灰蓝）
- 卡片背景：#1e293b（深蓝灰）
- 边框：#334155（中灰）
- 主色：#3b82f6（蓝色）
- 成功：#10b981（绿色）
- 警告：#f59e0b（橙色）
- 错误：#ef4444（红色）

### 布局设计
- 左侧：搜索面板（固定宽度280px）
- 中间：图谱画布（自适应宽度）
- 右侧：详情侧边栏（固定宽度360px，可折叠）
- 顶部：工具栏（全局）

### 交互设计
- 点击节点：弹出详情侧边栏
- 双击节点：展开邻居
- 拖拽节点：移动位置（图谱模式）/调整顺序（时序模式）
- 滚轮：缩放图谱
- 右键：上下文菜单

## 七、非功能需求

### 性能要求
- 首屏加载时间 < 2秒
- 图谱渲染支持500+节点流畅运行
- 侧边栏展开/收起动画 < 200ms

### 可访问性
- 支持键盘导航
- 支持屏幕阅读器
- 对比度符合WCAG标准

### 兼容性
- Chrome 90+
- Firefox 88+
- Safari 14+
