# LinkMind 测试指南

## 一、测试概述

本文档提供 LinkMind 知识图谱系统的测试指南，包括测试策略、测试用例、测试流程等内容。

## 二、测试环境

### 2.1 硬件要求

| 项目 | 要求 |
|------|------|
| CPU | 多核处理器，4核及以上 |
| 内存 | 8GB 及以上 |
| 磁盘 | 50GB 可用空间 |

### 2.2 软件环境

| 软件 | 版本 |
|------|------|
| Python | 3.12+ |
| Node.js | 20.x+ |
| Docker | 20.10+ |
| Neo4j | 5.26.0 |

### 2.3 测试工具

| 工具 | 用途 |
|------|------|
| pytest | Python 后端单元测试 |
| Jest | JavaScript 前端单元测试 |
| Cypress | 端到端测试 |
| Postman | API 接口测试 |

## 三、测试策略

### 3.1 测试类型

| 测试类型 | 说明 | 覆盖范围 |
|----------|------|----------|
| 单元测试 | 测试单个函数或组件 | 后端 API、前端组件 |
| 集成测试 | 测试多个模块协作 | API 调用链、组件交互 |
| 端到端测试 | 模拟真实用户操作 | 完整业务流程 |
| 性能测试 | 测试系统性能指标 | 响应时间、并发能力 |
| 安全测试 | 测试系统安全性 | 输入验证、权限控制 |

### 3.2 测试级别

| 级别 | 说明 | 执行时机 |
|------|------|----------|
| L0 - 单元测试 | 代码提交时自动执行 | CI 流水线 |
| L1 - 集成测试 | 合并到主分支前执行 | PR 审核 |
| L2 - 端到端测试 | 发布前执行 | 发布流程 |
| L3 - 性能测试 | 重大版本发布前执行 | 版本发布 |

## 五、测试流程

### 5.1 单元测试流程

```bash
# 后端单元测试
cd backend
uv run pytest src/tests/ -v

# 前端单元测试
cd frontend
npm test -- --coverage
```

### 5.2 端到端测试流程

```bash
# 启动开发服务器
cd frontend
npm run dev

# 运行 Cypress 测试
npx cypress run
```

### 5.3 API 接口测试

使用 Postman 或 curl 测试 API：

```bash
# 健康检查
curl http://localhost:8000/api/health

# 创建节点
curl -X POST http://localhost:8000/api/nodes \
  -H "Content-Type: application/json" \
  -d '{"name": "测试节点", "parent_id": "root"}'

# 获取节点列表
curl http://localhost:8000/api/nodes?parent_id=root
```

## 七、缺陷管理

### 7.1 缺陷分类

| 级别 | 说明 | 响应时间 |
|------|------|----------|
| P0 | 致命缺陷，系统无法运行 | 立即 |
| P1 | 严重缺陷，核心功能失效 | 24小时 |
| P2 | 一般缺陷，功能异常 | 3天 |
| P3 | 轻微缺陷，UI问题 | 7天 |

### 7.2 缺陷报告模板

```
【缺陷标题】简洁描述问题

【缺陷级别】P0/P1/P2/P3

【复现步骤】

2. 步骤二
3. 步骤三

【预期结果】描述期望的行为
```
