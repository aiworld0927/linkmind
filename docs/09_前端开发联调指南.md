# LinkMind 前端开发联调指南

## 一、开发环境配置

### 1.1 基础环境要求

| 环境 | 版本 | 说明 |
|------|------|------|
| Node.js | 20.x+ | 前端运行时 |
| npm | 10.x+ | 包管理器 |
| Python | 3.12+ | 后端运行时（联调需要） |
| Docker | 20.10+ | 数据库容器化 |

### 1.2 项目结构

```
frontend/
├── src/
│   ├── components/       # React组件
│   ├── store/            # Zustand状态管理
│   ├── types/            # TypeScript类型定义
│   ├── data/             # Mock数据
│   └── utils/            # 工具函数
├── package.json          # 依赖配置
├── vite.config.ts        # Vite配置
└── tsconfig.json         # TypeScript配置
```

### 1.3 安装依赖

```bash
cd frontend
npm install
```

## 二、开发模式启动

### 2.1 独立前端开发（Mock数据）

```bash
# 启动Vite开发服务器
npm run dev

# 访问地址
# http://localhost:5173/
```

此模式下，前端使用本地 `localStorage` 和 Mock 数据，无需启动后端服务。

### 2.2 前后端联调模式

#### 步骤一：启动 Neo4j 数据库

```bash
docker run -d \
  --name neo4j-dev \
  -p 7687:7687 \
  -p 7474:7474 \
  -e NEO4J_AUTH=neo4j/password \
  neo4j:5.26.0
```

#### 步骤二：启动后端服务

```bash
cd backend
uv pip install -e .
cd src
python main.py

# 访问地址
# http://localhost:8000/api/
```

#### 步骤三：启动前端服务

```bash
cd frontend
npm run dev

# 访问地址
# http://localhost:5173/
```

### 2.3 代理配置

Vite 自动代理 `/api` 请求到后端：

```typescript
// vite.config.ts
server: {
  proxy: {
    '/api': {
      target: 'http://localhost:8000',
      changeOrigin: true,
    },
  },
},
```

## 三、状态管理联调

### 3.1 Zustand Store 结构

```typescript
// src/store/index.ts
interface LinkMindStore {
  // 状态
  currentNodes: NodeItem[];
  currentLevelId: string;
  breadcrumbs: BreadItem[];
  globalDisplayMode: DisplayMode;
  searchKeyword: string;
  
  // Actions
  addNode: (node: NodeItem) => void;
  updateNode: (nodeId: string, patch: Partial<NodeItem>) => void;
  deleteNode: (nodeId: string, cascade: boolean) => void;
  drillDown: (targetNodeId: string) => void;
  backToBreadLevel: (targetIndex: number) => void;
}
```

### 3.2 使用 Store

```typescript
import { useStore } from '@/store';

function MyComponent() {
  const { currentNodes, addNode, deleteNode } = useStore();
  
  return (
    <div>
      {currentNodes.map(node => (
        <div key={node.id}>{node.name}</div>
      ))}
    </div>
  );
}
```

### 3.3 本地持久化

Store 自动同步到 `localStorage`：

```typescript
// src/store/index.ts
persist(
  (set, get) => ({ ... }),
  {
    name: 'linkmind-store',
  }
);
```

## 四、组件联调

### 4.1 核心组件说明

| 组件 | 职责 | 关键Props |
|------|------|-----------|
| GraphCanvas | X6图编辑画布 | onEditNode |
| Toolbar | 顶部工具栏 | - |
| Breadcrumbs | 面包屑导航 | - |
| NodeModal | 节点新增/编辑弹窗 | visible, onCancel, editingNode |
| NodePopover | 悬浮弹窗展示 | nodeId, x, y, onClose |
| NodeDrawer | 右侧抽屉展示 | - |
| NoteList | 笔记列表 | items, parentNodeId, onAdd, onEdit, onDelete |
| ListItemModal | 笔记新增/编辑弹窗 | visible, onCancel, onSubmit, editingItem |

### 4.2 组件通信

```
Toolbar ──(新增/搜索)──→ Store
Breadcrumbs ──(回退)──→ Store
GraphCanvas ──(下钻/展开)──→ Store
NodeModal ──(提交)──→ Store
NoteList ──(增删改)──→ Store
Store ──(状态变化)──→ 所有订阅组件
```

### 4.3 自定义事件处理

在 `GraphCanvas` 中注册 X6 事件：

```typescript
graph.on('node:click', handleNodeClick);
graph.on('node:dblclick', handleNodeDoubleClick);
graph.on('blank:click', () => {
  closePopover();
  closeDrawer();
});
```

## 五、API 联调

### 5.1 当前数据流向

```
前端 Store (localStorage) ←→ 用户操作
                          ↓
                   数据导入导出
                          ↓
                    JSON 文件
```

### 5.2 后端 API 集成

当需要持久化到 Neo4j 时，调用后端 API：

```typescript
// 示例：保存节点到后端
async function saveNodeToBackend(node: NodeItem) {
  const response = await fetch('/api/nodes', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
    },
    body: JSON.stringify(node),
  });
  return response.json();
}
```

### 5.3 API 接口列表

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/nodes` | GET | 获取节点列表 |
| `/api/nodes` | POST | 创建节点 |
| `/api/nodes/{id}` | GET | 获取单个节点 |
| `/api/nodes/{id}` | PUT | 更新节点 |
| `/api/nodes/{id}` | DELETE | 删除节点 |
| `/api/notes` | POST | 创建笔记 |
| `/api/notes` | GET | 获取笔记列表 |
| `/api/notes/{id}` | PUT | 更新笔记 |
| `/api/notes/{id}` | DELETE | 删除笔记 |
| `/api/health` | GET | 健康检查 |

### 5.4 错误处理

```typescript
try {
  const response = await fetch('/api/nodes');
  if (!response.ok) {
    throw new Error('API请求失败');
  }
  const data = await response.json();
} catch (error) {
  message.error('请求失败，请检查网络连接');
}
```

## 七、代码规范

### 7.1 TypeScript 规范

- 使用 `interface` 定义类型
- 使用 `type` 定义联合类型
- 为所有函数和组件添加类型注解
- 使用 `any` 时需谨慎

### 7.2 组件规范

- 组件名使用 PascalCase
- 文件命名使用 PascalCase
- 使用函数组件 + Hooks
- 避免在组件内部定义函数（使用 `useCallback`）

### 7.3 样式规范

- 使用内联样式（当前项目）或 CSS Modules
- 颜色使用统一变量
- 间距使用 4px 倍数
- 字体大小使用 12px/14px/16px/18px

### 7.4 代码格式化

```bash
# 格式化代码
npm run format

# 代码检查
npm run lint

# 类型检查
npm run typecheck
```

## 九、开发流程

### 9.1 新增功能

1. 在 `src/types/index.ts` 定义类型
2. 在 `src/store/index.ts` 添加状态和 Action
3. 创建或修改组件
4. 添加 Mock 数据
5. 测试功能
6. 编写测试用例

### 9.2 修复 Bug

1. 复现问题
2. 定位代码位置
3. 分析原因
4. 修复代码
5. 验证修复
6. 编写回归测试

### 9.3 代码提交
